# Architecture Overview

Nezumi UI uses a small, source-first package architecture. The current mockup does not split tokens, layout, hooks, or schemas into separate packages. The UI package owns the public API and the token CSS.

## Layers

```
Applications
  import public leaf exports
@nezumi/ui package
  src/components/<name>.tsx     # public surface
  src/layout/index.ts           # public layout entrypoint
  src/lib/utils.ts              # public utility entrypoint
Internal implementation
  src/atoms/
CSS foundation
  src/styles/global.css
  src/styles/tokens/*.css
  src/styles/semantic/*.css
  src/styles/components/button.css
```

## Public API

The public API is granular:

```tsx
import { Button } from "@nezumi/ui/components/button"
import { Grid } from "@nezumi/ui/layout"
import { cn } from "@nezumi/ui/lib/utils"
```

The root package export is intentionally absent. This keeps dependency graphs small and matches the shadcn monorepo model.

The package exports TypeScript source files directly. This follows the
source-first / Just-in-Time package model: app bundlers compile the UI package as
part of the app build, while package-local checks use `tsc --noEmit`.

## Token Architecture

Tailwind CSS 4.2.4 reads tokens from `@theme`. Semantic color names follow **[`DESIGN.md`](../../../DESIGN.md) §2** (e.g. `--color-surface`, `--color-text`, `--color-brand`); primitives are **`--color-nezumi-*`** in `tokens/colors.css` (`@theme`, OKLCH from DESIGN hex).

```css
@theme {
  --color-surface: var(--color-nezumi-bg);
  --color-text: var(--color-nezumi-koi);
  --spacing-16: 1rem;
  --radius-lg: 0.5rem;
}
```

Then components consume generated utilities:

```tsx
<div className="bg-surface text-text p-16 rounded-lg" />
```

## Framework Baseline

- React 19.2.6 & react-dom 19.2.6: refs are regular props in function components; `<ViewTransition>`, `useEffectEvent`, and `<Activity>` are available for app-level interaction work.
- Next.js 16.2.5: App Router, `proxy.ts` for request interception, Turbopack as the default bundler, `cacheComponents`, and `"use cache"` are the documented defaults for new app architecture.
- Tailwind CSS 4.2.4: CSS-first configuration with `@theme`.
- Radix UI: unified `radix-ui` package.
- shadcn/ui: granular package exports and `components.json`.

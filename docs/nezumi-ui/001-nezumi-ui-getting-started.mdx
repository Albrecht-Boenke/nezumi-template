# Getting Started

## Install And Verify

From the workspace root:

```bash
pnpm install
next build apps/homepage
next build apps/members
next build apps/operations
next build apps/playground
```

`@packages/ui` is an internal source-first package. It does not expose a
package-local build script in the root workflow; consuming apps transpile the
exported source files through their Next.js build.

The workspace uses pnpm catalogs for shared framework versions. **Pins in this repo:** `react` & `react-dom` **19.2.6**, `typescript` **6.0.3**, `tailwindcss` & `@tailwindcss/postcss` **4.2.4**, `next` **16.2.5** (app); **pnpm** **10.33.3**; **Node** **24.x** (`^24.0.0`) — see root [`FRAMEWORK_VERSION_REFERENCES.md`](../../FRAMEWORK_VERSION_REFERENCES.md).

## Import Styles

Apps own the Tailwind entry and import the shared tokens after declaring the
source paths Tailwind must scan:

```css
@import "tailwindcss";
@custom-variant dark (&:where(.dark, .dark *));

@source "../../../packages/ui/src";
@source "../";

@import "@packages/ui/design-tokens.css";
```

Use `@packages/ui/globals.css` only for one-shot environments such as Storybook
where the UI package owns the complete Tailwind entry.

## Import Components

Use public leaf exports:

```tsx
import { Button } from "@packages/ui/components/button"
import { Container } from "@packages/ui/components/container"
import { Flex } from "@packages/ui/components/flex"

export function Example() {
  return (
    <Container px="24" py="32">
      <Flex gap="16" align="center">
        <Button>Save</Button>
      </Flex>
    </Container>
  )
}
```

Avoid importing from `atoms`, `molecules`, `organisms`, or `templates` in application code. Those folders are internal implementation detail.

## Use Tokens

Use Tailwind utilities generated from `@theme`:

```tsx
<div className="bg-surface text-text p-16 rounded-lg" />
```

Use CSS variables only when runtime CSS needs a raw value:

```css
.panel {
  background: var(--color-surface);
  padding: var(--spacing-16);
}
```

# Monorepo Architecture

**Tech Stack:** Next.js **16.2.5**, React **19.2.6**, react-dom **19.2.6**, TypeScript **6.0.3**, Tailwind CSS **4.2.4**, Radix UI, shadcn-compatible registry tooling, pnpm **10.33.3**, Node **24.x** (`^24.0.0`).

## Workspace

```
.
├── package.json
├── pnpm-workspace.yaml
└── packages/
    └── ui/
        ├── components.json
        ├── package.json
        ├── tsconfig.json
        └── src/
```

## UI Package Contract

Consumer imports:

```tsx
import { Button } from "@packages/ui/components/button"
import { Container, Grid } from "@packages/ui/layout"
import { cn } from "@packages/ui/lib/utils"
```

Internal folders:

```
src/atoms/
```

Public folders:

```
src/components/
src/layout/
src/lib/
src/styles/
```

## Server Components

For Next.js 16.2.5 and React 19.2.6 / react-dom 19.2.6, keep server and client boundaries explicit. Use `proxy.ts` for request interception, Cache Components with `cacheComponents`, and `"use cache"` only where stable cached output is intended.

## Export Pattern

Use package-map exports instead of a root barrel. `@packages/ui` is an internal
source-first package, so public entries point at source files and app builds
transpile them through `transpilePackages: ["@packages/ui"]`:

```json
{
  "imports": {
    "#components/*": "./src/components/*.tsx",
    "#lib/*": "./src/lib/*.ts"
  },
  "exports": {
    "./globals.css": "./src/styles/global.css",
    "./design-tokens.css": "./src/styles/design-tokens.css",
    "./components/button": "./src/components/button.tsx",
    "./layout": "./src/layout/index.ts",
    "./lib/utils": "./src/lib/utils.ts"
  }
}
```

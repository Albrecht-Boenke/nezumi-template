# Foundation And Design Tokens

The foundation layer lives inside `packages/ui/src/styles/`. It is CSS-first and Tailwind CSS 4.2.4 native. **Color naming follows [`DESIGN.md`](../../../DESIGN.md) §2:** primitives are registered in `tokens/colors.css` inside **`@theme`** as `--color-nezumi-*` with concrete OKLCH coordinates converted from the DESIGN hex table. Semantic `--color-*` live in `semantic/colors.css` (`@theme`). Dark overrides: `design-tokens.css` (`.dark`).

## Token Definition

Primitives (concrete OKLCH, `@theme` — hex in [`DESIGN.md`](../../../DESIGN.md) remains the authoring reference):

```css
@theme {
  --color-nezumi-sabi: oklch(0.4475 0.0220 213.59);
  --color-nezumi-paper: oklch(0.9818 0.0054 95.10);
  /* … see tokens/colors.css */
}
```

Semantic colors register Tailwind utilities:

```css
@theme {
  --color-brand: var(--color-nezumi-sabi);
  --color-text: var(--color-nezumi-koi);
  --color-surface: var(--color-nezumi-bg);
  --spacing-16: 1rem;
  --text-sm: 0.875rem;
  --radius-lg: 0.5rem;
  --duration-normal: 200ms;
}
```

`@theme` registers utilities with Tailwind:

```tsx
<div className="bg-surface text-text p-16 rounded-lg duration-normal" />
```

## Dark Mode

Dark-mode overrides belong under `.dark` in `design-tokens.css`, not inside the light `@theme` block.

```css
.dark {
  --color-brand: var(--color-nezumi-minato);
  --color-text: var(--color-nezumi-kinu);
  --color-surface: var(--color-nezumi-dark-bg);
  color-scheme: dark;
}
```

## Rule

Do not create parallel TypeScript token objects. CSS `@theme` and the shared CSS imports are the source of truth.
